# Investment Tracker — Obsidian向けプライベート・ポートフォリオトラッカー

[English](https://github.com/joelam2023/investment-tracker/blob/main/README.md) | [简体中文](https://github.com/joelam2023/investment-tracker/blob/main/README.zh-CN.md) | [繁體中文](https://github.com/joelam2023/investment-tracker/blob/main/README.zh-TW.md) | 日本語 | [한국어](https://github.com/joelam2023/investment-tracker/blob/main/README.ko.md) | [Español](https://github.com/joelam2023/investment-tracker/blob/main/README.es.md) | [Deutsch](https://github.com/joelam2023/investment-tracker/blob/main/README.de.md) | [Français](https://github.com/joelam2023/investment-tracker/blob/main/README.fr.md) | [Português (Brasil)](https://github.com/joelam2023/investment-tracker/blob/main/README.pt-BR.md)

**あなたのポートフォリオ。あなたのVault。暗号化して保管。**

Investment Trackerは、Obsidian向けのプライバシーを重視したローカルファーストのポートフォリオトラッカーです。アカウント登録、テレメトリ、開発者が運用するバックエンドを必要とせず、暗号化された投資記録を自分のVault内に保持したまま、キャッシュフロー、資産評価額、収益率、ベンチマーク実績を追跡できます。

口座単位で動作するため、銘柄ごとの取引履歴を管理しなくても投資パフォーマンスを計算できます。

## 主な仕様

| 項目 | Investment Trackerの仕組み |
| --- | --- |
| 投資記録 | 暗号化してユーザー自身のObsidian Vault内に保存 |
| 開発者が運用するバックエンド | なし |
| アカウントまたはサインイン | 不要 |
| テレメトリと分析 | なし |
| 暗号化 | AES-256-GCM。台帳キーはPBKDF2-SHA256と別個の回復キーで保護 |
| オプションのネットワークアクセス | 自動ベンチマークモードでFREDから公開ベンチマークおよび為替データを取得 |
| Vaultの同期 | Obsidian SyncやiCloudなど、ユーザーが選択したサービスが暗号化された台帳を同期する場合あり |
| エクスポート | ユーザーが作成するJSONおよびCSVエクスポートは平文 |

## 機能

- USD、GBP、SGD、CNY、TWD、JPY、KRW、EUR、BRLによる複数の投資口座。
- 入金、出金、資産評価額を記録する、イミュータブルなイベントベースの記帳方式。
- XIRR、累積損益、年次収益率、月次Modified Dietz収益率。
- 同一キャッシュフローを用いたS&P 500 Price Indexとの比較。
- 為替のクオート方向を明示的に検証する、通貨対応のFREDベンチマーク換算。
- パスワードロック、独立した回復キー、金額の非表示、設定可能な自動ロック。
- 暗号化されたJSONイベントをユーザー自身のVault内に保存。
- 明示的な操作によるローカルJSONおよびCSVエクスポート。設定画面の操作ではパスワードによる再認証が必要。
- インターフェース言語の自動選択、手動による変更、英語へのフォールバック。
- 英語、簡体字中国語、繁体字中国語、日本語、韓国語、スペイン語、ドイツ語、フランス語、ブラジルポルトガル語。

## このような方に適しています

- ポートフォリオ記録を自分のObsidian Vault内に保存したい、プライバシーを重視する投資家。
- 口座単位の入金、出金、資産評価額を手動で記録する方。
- XIRR、月次・年次パフォーマンス、S&P 500との比較を確認したい投資家。
- 新たな金融サービスのアカウントを作らず、ローカルファーストのワークフローを使いたい方。

## 対象外の用途

- 証券口座との同期。
- リアルタイムの保有銘柄・価格情報、税務上のロット管理、自動取引。
- 証券会社の取引報告書、税務記録、専門家による金融アドバイスの代替。
- 端末が侵害された場合や、別の悪意あるプラグインが存在する場合に、ロック解除中のVaultを保護すること。

## インストールとアップデート

**Obsidian → 設定 → コミュニティプラグイン → 閲覧**から**Investment Tracker**をインストールします。「Investment Tracker」を検索してプラグインを選択し、**インストール**、続いて**有効化**を選択してください。

アップデートはObsidianのコミュニティプラグイン更新機能を通じて提供されます。

手動インストールまたはテストを行う場合は、`main.js`、`manifest.json`、`styles.css`を次の場所に配置します。

```text
<Vault>/.obsidian/plugins/investment-tracker/
```

## 基本的な使い方

1. リボンからInvestment Trackerを開きます。
2. パスワードを設定し、生成された回復キーをVaultの外部に保存します。
3. 口座を作成し、初回の資産評価額を記録します。
4. 外部からの入金、外部への出金、更新後の口座総評価額を記録します。
5. 目のボタンで金額を表示または非表示にします。
6. 月次・年次収益率を確認し、選択したベンチマークと比較します。
7. **設定 → Investment Tracker → プライバシーと暗号化**で自動ロックのルールを選択します。

インターフェース言語を変更しても、既存口座の通貨は変更されません。新規インストール時には、ロケール情報は初期通貨の提案にのみ使用されます。口座を作成する前に変更できます。

## プライバシーとセキュリティ

Investment Trackerには、開発者が運用するクラウド、アカウントシステム、テレメトリ、分析、広告、自動アップロード機能はありません。口座名、日付、金額、メモ、イベントデータは暗号化され、ユーザー自身のObsidian Vault内に保存されます。新規インストールでは`Investment Tracker Data`フォルダーを使用し、アップデート時には既存の安全なデータパスを維持します。

イベントデータはAES-256-GCMで暗号化されます。台帳キーは、パスワードから導出したPBKDF2-SHA256キーと別個の回復キーを使ってラップされます。パスワードとラップ解除後の台帳キーがプラグイン設定に書き込まれることはありません。

自動ロックには独立した2つのルールがあります。Investment Trackerから離れたとき、またはObsidianがフォーカスを失ったときに即時ロックするルールと、Investment Trackerで1分、5分、15分、30分のいずれかの時間、操作がなかった場合にロックするルールです。少なくとも一方のルールは常に有効です。離れたときの即時ロックを無効にしている場合でも、離れると金額が非表示になり、展開した履歴が折りたたまれ、機密性の高いダイアログが閉じられます。操作なしルールが作動したとき、または手動でロックしたときに、台帳キーがメモリから消去されます。

新しく生成された回復キーは画面を離れると非表示になり、台帳のロックを再度解除した後にのみ表示されます。回復キーはVaultの外部に保管し、強力で固有のパスワードを使用してください。

暗号化は、保存された台帳ファイルが不用意に漏れることを防ぎます。ただし、プラグインのロック解除中のデータ、侵害された端末、スクリーンショットやクリップボードからの漏えい、同じVaultにアクセスできる別の悪意あるプラグインからデータを保護することはできません。

### 同期とエクスポート

Investment Trackerは同期サービスを運用していません。ユーザーがObsidian Sync、iCloud、その他のVault同期サービスを有効にすると、ユーザーが選択したそのサービスが暗号化された台帳ファイルを端末間で同期する場合があります。

JSONおよびCSVエクスポートは、ユーザーが明示的にエクスポートした場合にのみ作成される平文ファイルです。エクスポートファイルは機密性の高い財務記録として扱い、適切に保管または削除してください。

詳細は[プライバシーポリシー](https://github.com/joelam2023/investment-tracker/blob/main/PRIVACY.md)と[セキュリティポリシー](https://github.com/joelam2023/investment-tracker/blob/main/SECURITY.md)をお読みください。

## ネットワーク通信について

基本的な記帳と収益率計算に、開発者が運用するサービスは必要ありません。自動ベンチマークモードは、S&P 500および通貨換算データを取得するため、Federal Reserve Economic Dataサービスの`fred.stlouisfed.org`にHTTPS GETリクエストを送信します。

リクエストに含まれるのは、公開系列ID、為替系列の選択に必要な通貨、日付範囲だけです。口座名、残高、キャッシュフローの金額、資産評価額、メモ、パスワード、回復キー、台帳の内容は含まれません。

自動FREDリクエストを避けるには、手動ベンチマークモードを選択できます。自動ベンチマーク更新にはインターネット接続が必要です。プラグインが使用するS&P 500系列は価格指数であり、配当は含まれません。

## よくある質問

### Investment Trackerは私のポートフォリオデータをアップロードしますか？

ポートフォリオ台帳が開発者運用のバックエンドへ送信されることはありません。プラグインには、開発者のアカウントシステム、テレメトリ、分析、自動ポートフォリオアップロード機能はありません。自動ベンチマークモードでは、[ネットワーク通信について](#ネットワーク通信について)で説明した限定的なFREDリクエストのみを行います。

### 投資データはどこに保存されますか？

暗号化された台帳はユーザー自身のObsidian Vault内に保存されます。新規インストールでは`Investment Tracker Data`を使用します。ユーザーが選択したサービスでVaultを同期する場合、そのサービスが暗号化された台帳を保存または転送する場合もあります。

### 投資データは暗号化されていますか？

保存されたイベントデータはAES-256-GCMで暗号化されます。パスワードから導出したPBKDF2-SHA256キーと別個の回復キーが台帳キーを保護します。プラグインのロック解除中はデータが表示可能であり、ユーザーが作成するJSONまたはCSVエクスポートは暗号化されません。

### Investment Trackerはオフラインで使えますか？

ローカルの記録と収益率計算は、開発者が運用するサービスなしで使用できます。FREDのベンチマークと通貨の自動更新にはインターネット接続が必要です。手動ベンチマークモードでは、これらのリクエストを行いません。

### 証券口座に接続しますか？

いいえ。Investment Trackerは証券口座に接続しません。外部からの入金、外部への出金、口座総評価額をユーザーが手動で記録します。

### 個別の保有銘柄や取引を追跡しますか？

銘柄ごとの取引履歴は不要です。このプラグインはリアルタイムの保有銘柄や税務上のロット管理ではなく、口座単位のキャッシュフローと資産評価額を管理するために設計されています。

### FREDにはどのような情報が送信されますか？

自動ベンチマークのリクエストに含まれるのは、公開系列ID、為替系列の選択に必要な通貨、日付範囲だけです。ポートフォリオ記録や認証情報は含まれません。

### パスワードを忘れた場合はどうなりますか？

プラグインの回復手順に従い、別に保管した回復キーを使用してアクセスを回復してください。パスワードと回復キーの両方を失うと、暗号化された台帳にアクセスできなくなる可能性があります。

### JSONとCSVのエクスポートは暗号化されますか？

いいえ。JSONとCSVのエクスポートは平文です。機密性の高い財務記録として取り扱ってください。

## ヘルプとフィードバック

**設定 → Investment Tracker → ヘルプとフィードバック**を開き、バグの報告、機能の提案、機密情報を含まない診断情報のコピーを行えます。どの言語でも報告できます。

ユーザーがボタンをクリックした場合に限り、フィードバックリンクからGitHubが開きます。プラグインが自動的に報告を作成したり、台帳データ、口座名、残高、取引、パスワード、回復キー、Vault名、Vaultのパス、診断情報を開発者へ送信したりすることはありません。送信前にコピーした診断情報を確認し、スクリーンショットに含まれる機密情報を削除してください。

セキュリティまたはプライバシー上の脆弱性は、公開Issueではなく[GitHubの非公開脆弱性報告](https://github.com/joelam2023/investment-tracker/security/advisories/new)から報告してください。

## 開発

```bash
npm ci
npm run check
npm run build:release
npm run privacy:check
```

翻訳には英語の原文をフォールバックとして使用します。ユーザー向けテキストを変更するPull Requestでは、すべてのロケールを更新し、補間プレースホルダーを変更しないでください。

リリースタグは、`v`を付けず、`manifest.json`のセマンティックバージョンと完全に一致させる必要があります。リリースワークフローは、公開前の手動確認用に、`main.js`、`manifest.json`、`styles.css`のみを含むGitHub Releaseのドラフトを作成します。

メンテナー向けの手順については、[リリースガイド](https://github.com/joelam2023/investment-tracker/blob/main/RELEASING.md)をご覧ください。

## 金融に関する免責事項

このプラグインは記録・計算ツールであり、金融、税務、法律、投資に関する助言を提供するものではありません。意思決定を行う前に、重要な計算結果を独自に検証してください。
