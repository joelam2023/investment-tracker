# Investment Tracker — Obsidian용 비공개 포트폴리오 추적기

[English](https://github.com/joelam2023/investment-tracker/blob/main/README.md) | [简体中文](https://github.com/joelam2023/investment-tracker/blob/main/README.zh-CN.md) | [繁體中文](https://github.com/joelam2023/investment-tracker/blob/main/README.zh-TW.md) | [日本語](https://github.com/joelam2023/investment-tracker/blob/main/README.ja.md) | 한국어 | [Español](https://github.com/joelam2023/investment-tracker/blob/main/README.es.md) | [Deutsch](https://github.com/joelam2023/investment-tracker/blob/main/README.de.md) | [Français](https://github.com/joelam2023/investment-tracker/blob/main/README.fr.md) | [Português (Brasil)](https://github.com/joelam2023/investment-tracker/blob/main/README.pt-BR.md)

**내 포트폴리오. 내 Vault. 암호화된 기록.**

Investment Tracker는 개인정보 보호를 중시하는 Obsidian용 로컬 우선 포트폴리오 추적기입니다. 계정 가입, 원격 측정, 개발자가 운영하는 백엔드 없이 암호화된 투자 기록을 자신의 Vault에 보관하면서 현금 흐름, 자산 평가액, 수익률 및 벤치마크 성과를 추적할 수 있습니다.

계좌 단위로 작동하므로 종목별 거래 내역을 관리하지 않고도 투자 성과를 계산할 수 있습니다.

## 주요 정보

| 항목 | Investment Tracker의 작동 방식 |
| --- | --- |
| 투자 기록 | 암호화하여 사용자의 Obsidian Vault 안에 저장 |
| 개발자가 운영하는 백엔드 | 없음 |
| 계정 또는 로그인 | 필요 없음 |
| 원격 측정 및 분석 | 없음 |
| 암호화 | AES-256-GCM, PBKDF2-SHA256 및 별도의 복구 키로 원장 키 보호 |
| 선택적 네트워크 접속 | 자동 벤치마크 모드에서 FRED의 공개 벤치마크 및 환율 데이터 요청 |
| Vault 동기화 | Obsidian Sync나 iCloud처럼 사용자가 선택한 서비스에서 암호화된 원장을 동기화할 수 있음 |
| 내보내기 | 사용자가 만든 JSON 및 CSV 내보내기 파일은 평문 |

## 기능

- USD, GBP, SGD, CNY, TWD, JPY, KRW, EUR 또는 BRL로 여러 투자 계좌 관리.
- 계좌를 만들 때 통화를 선택하고 이후 이름이나 비교 벤치마크를 편집할 수 있으며, 암호화된 기록을 삭제하지 않고 계좌를 보관하거나 복원할 수 있습니다.
- 입금, 출금 및 자산 평가액을 위한 변경 불가능한 이벤트 기반 장부 기록.
- XIRR, 누적 손익, 연간 수익률 및 월간 Modified Dietz 수익률 계산.
- 동일 현금 흐름을 적용한 S&P 500 Price Index 비교.
- 환율 표시 방향을 명시적으로 확인하는 통화별 FRED 벤치마크 환산.
- 비밀번호 잠금, 별도의 복구 키, 재무 수치 숨김 및 설정 가능한 자동 잠금.
- 암호화된 JSON 이벤트를 사용자의 Vault 안에 저장.
- 사용자가 명시적으로 실행하는 로컬 JSON 및 CSV 내보내기. 설정 화면에서는 비밀번호 재인증 필요.
- 인터페이스 언어 자동 선택, 수동 변경 및 영어 대체 언어 지원.
- 영어, 중국어 간체, 중국어 번체, 일본어, 한국어, 스페인어, 독일어, 프랑스어 및 브라질 포르투갈어 지원.

## 이런 분께 적합합니다

- 포트폴리오 기록을 자신의 Obsidian Vault 안에 두고 싶은 개인정보 보호 중심의 투자자.
- 계좌별 입금, 출금 및 자산 평가액을 수동으로 기록하는 사용자.
- XIRR, 월간·연간 성과 및 S&P 500 비교를 확인하고 싶은 투자자.
- 별도의 금융 계정을 만들지 않고 로컬 우선 작업 방식을 선호하는 사용자.

## 다음 용도로는 설계되지 않았습니다

- 증권사 계좌 동기화.
- 실시간 보유 자산, 시세 피드, 세금 단위 원가 관리 또는 자동 매매.
- 증권사 명세서, 세금 기록 또는 전문가의 금융 자문 대체.
- 기기가 침해되었거나 다른 악성 플러그인이 있는 상황에서 잠금 해제된 Vault 보호.

## 설치 및 업데이트

**Obsidian → 설정 → 커뮤니티 플러그인 → 탐색**에서 **Investment Tracker**를 설치합니다. “Investment Tracker”를 검색해 플러그인을 선택하고 **설치**, **활성화**를 차례로 누르세요.

업데이트는 Obsidian의 커뮤니티 플러그인 업데이트 기능을 통해 제공됩니다.

수동 설치 또는 테스트를 하려면 `main.js`, `manifest.json`, `styles.css`를 다음 위치에 넣으세요.

```text
<Vault>/.obsidian/plugins/investment-tracker/
```

## 기본 사용법

1. 리본에서 Investment Tracker를 엽니다.
2. 비밀번호를 설정하고 생성된 복구 키를 Vault 외부에 보관합니다.
3. 계좌를 만들고 초기 자산 평가액을 기록합니다.
4. 외부 입금, 외부 출금 및 갱신된 계좌 총자산 평가액을 기록합니다.
5. 눈 모양 버튼으로 재무 수치를 표시하거나 숨깁니다.
6. 월간 및 연간 수익률을 확인하고 선택한 벤치마크와 비교합니다.
7. **설정 → Investment Tracker → 개인정보 보호 및 암호화**에서 자동 잠금 규칙을 선택합니다.

인터페이스 언어를 변경해도 기존 계좌의 통화는 변경되지 않습니다. 새로 설치할 때 플러그인은 로캘 정보를 초기 통화 제안에만 사용하며, 계좌를 만들기 전에 사용자가 이를 변경할 수 있습니다.

## 개인정보 보호 및 보안

Investment Tracker에는 개발자가 운영하는 클라우드, 계정 시스템, 원격 측정, 분석, 광고 또는 자동 업로드 기능이 없습니다. 계좌 이름, 날짜, 금액, 메모 및 이벤트 데이터는 암호화되어 사용자의 Obsidian Vault 안에 저장됩니다. 새로 설치하면 `Investment Tracker Data` 폴더를 사용하며, 업데이트 시 안전한 기존 데이터 경로를 유지합니다.

이벤트 데이터는 AES-256-GCM으로 암호화됩니다. 원장 키는 비밀번호에서 파생된 PBKDF2-SHA256 키와 별도의 복구 키를 사용해 래핑됩니다. 비밀번호와 래핑이 해제된 원장 키는 플러그인 설정에 기록되지 않습니다.

자동 잠금에는 서로 독립된 두 가지 규칙이 있습니다. Investment Tracker를 벗어나거나 Obsidian이 포커스를 잃으면 즉시 잠그는 규칙과 Investment Tracker에서 1분, 5분, 15분 또는 30분 동안 활동이 없으면 잠그는 규칙입니다. 최소한 하나의 규칙은 계속 활성화됩니다. 화면을 벗어날 때 즉시 잠그는 기능을 끄더라도 재무 수치는 숨겨지고, 펼친 기록은 접히며, 민감한 대화 상자는 닫힙니다. 비활동 규칙이 작동하거나 수동으로 잠그면 원장 키가 메모리에서 지워집니다.

새로 생성된 복구 키는 화면을 벗어나면 숨겨지고, 원장의 잠금을 다시 해제한 뒤에만 표시됩니다. 복구 키는 Vault 외부에 보관하고 강력하고 고유한 비밀번호를 사용하세요.

암호화는 저장된 원장 파일이 우연히 노출되는 것을 방지합니다. 그러나 플러그인 잠금이 해제된 동안의 데이터, 침해된 기기, 스크린샷이나 클립보드를 통한 노출 또는 같은 Vault에 접근할 수 있는 다른 악성 플러그인으로부터는 보호할 수 없습니다.

### 동기화 및 내보내기

Investment Tracker는 동기화 서비스를 운영하지 않습니다. 사용자가 Obsidian Sync, iCloud 또는 다른 Vault 동기화 서비스를 활성화하면 사용자가 선택한 해당 서비스가 기기 간에 암호화된 원장 파일을 동기화할 수 있습니다.

JSON 및 CSV 내보내기는 사용자가 명시적으로 내보낼 때만 생성되는 평문 파일입니다. 내보낸 파일을 민감한 금융 기록으로 취급하고 적절히 보관하거나 삭제하세요.

전체 [개인정보 처리방침](https://github.com/joelam2023/investment-tracker/blob/main/PRIVACY.md) 및 [보안 정책](https://github.com/joelam2023/investment-tracker/blob/main/SECURITY.md)을 읽어보세요.

## 네트워크 공개 사항

핵심 장부 기록 및 수익률 계산에는 개발자가 운영하는 서비스가 필요하지 않습니다. 자동 벤치마크 모드는 S&P 500 및 통화 환산 데이터를 가져오기 위해 Federal Reserve Economic Data 서비스의 `fred.stlouisfed.org`로 HTTPS GET 요청을 보냅니다.

이 요청에는 공개 시계열 식별자, 환율 시계열 선택에 필요한 통화 및 날짜 범위만 포함됩니다. 계좌 이름, 잔액, 현금 흐름 금액, 자산 평가액, 메모, 비밀번호, 복구 키 또는 원장 내용은 포함되지 않습니다.

자동 FRED 요청을 피하려면 수동 벤치마크 모드를 선택할 수 있습니다. 자동 벤치마크 업데이트에는 인터넷 연결이 필요합니다. 플러그인에서 사용하는 S&P 500 시계열은 가격 지수이며 배당금은 포함하지 않습니다.

## 자주 묻는 질문

### Investment Tracker가 내 포트폴리오 데이터를 업로드하나요?

포트폴리오 원장은 개발자가 운영하는 백엔드로 전송되지 않습니다. 플러그인에는 개발자 계정 시스템, 원격 측정, 분석 또는 포트폴리오 자동 업로드 기능이 없습니다. 자동 벤치마크 모드는 [네트워크 공개 사항](#네트워크-공개-사항)에 설명된 제한적인 FRED 요청만 보냅니다.

### 투자 데이터는 어디에 저장되나요?

암호화된 원장은 사용자의 Obsidian Vault 안에 저장됩니다. 새로 설치하면 `Investment Tracker Data`를 사용합니다. 사용자가 선택한 서비스로 Vault를 동기화하면 해당 서비스가 암호화된 원장을 저장하거나 전송할 수도 있습니다.

### 투자 데이터가 암호화되나요?

저장된 이벤트 데이터는 AES-256-GCM으로 암호화됩니다. 비밀번호에서 파생된 PBKDF2-SHA256 키와 별도의 복구 키가 원장 키를 보호합니다. 플러그인 잠금이 해제된 동안에는 데이터를 볼 수 있으며, 사용자가 만든 JSON 또는 CSV 내보내기 파일은 암호화되지 않습니다.

### Investment Tracker를 오프라인에서 사용할 수 있나요?

로컬 기록과 수익률 계산은 개발자가 운영하는 서비스 없이 사용할 수 있습니다. FRED 벤치마크 및 통화 자동 업데이트에는 인터넷 연결이 필요하며, 수동 벤치마크 모드에서는 이러한 요청을 보내지 않습니다.

### 증권사 계좌에 연결되나요?

아니요. Investment Tracker는 증권사 계좌에 연결되지 않습니다. 사용자가 외부 입금, 외부 출금 및 계좌 총자산 평가액을 직접 기록합니다.

### 개별 보유 자산이나 거래를 추적하나요?

종목별 거래 내역은 필요하지 않습니다. 이 플러그인은 실시간 보유 자산이나 세금 단위 원가 관리가 아니라 계좌별 현금 흐름과 자산 평가액을 기록하도록 설계되었습니다.

### FRED로 어떤 정보가 전송되나요?

자동 벤치마크 요청에는 공개 시계열 식별자, 환율 시계열 선택에 필요한 통화 및 날짜 범위만 포함됩니다. 포트폴리오 기록과 인증 정보는 포함되지 않습니다.

### 비밀번호를 잊으면 어떻게 되나요?

플러그인의 복구 절차에 따라 별도로 보관한 복구 키를 사용해 다시 접근하세요. 비밀번호와 복구 키를 모두 분실하면 암호화된 원장에 접근하지 못할 수 있습니다.

### JSON 및 CSV 내보내기 파일도 암호화되나요?

아니요. JSON 및 CSV 내보내기 파일은 평문이므로 민감한 금융 기록으로 취급해야 합니다.

## 도움말 및 피드백

**설정 → Investment Tracker → 도움말 및 피드백**을 열어 버그를 신고하거나 기능을 제안하고, 민감한 정보가 없는 진단 정보를 복사할 수 있습니다. 어떤 언어로든 신고할 수 있습니다.

피드백 링크는 사용자가 버튼을 클릭한 뒤에만 GitHub를 엽니다. 플러그인이 자동으로 신고를 만들거나 원장 데이터, 계좌 이름, 잔액, 거래, 비밀번호, 복구 키, Vault 이름, Vault 경로 또는 진단 정보를 개발자에게 보내지 않습니다. 제출 전에 복사한 진단 정보를 검토하고 스크린샷에서 민감한 내용을 삭제하세요.

보안 또는 개인정보 보호 취약점은 공개 Issue가 아닌 [GitHub 비공개 취약점 신고](https://github.com/joelam2023/investment-tracker/security/advisories/new)를 통해 알려주세요.

## 개발

```bash
npm ci
npm run check
npm run build:release
npm run privacy:check
```

번역은 영어 원문 문자열을 대체 언어로 사용합니다. 사용자에게 표시되는 텍스트를 변경하는 Pull Request는 모든 로캘을 업데이트하고 보간 자리표시자를 변경하지 않아야 합니다.

릴리스 태그는 `v` 접두사 없이 `manifest.json`의 시맨틱 버전과 정확히 일치해야 합니다. 릴리스 작업 흐름은 게시 전 수동 검토를 위해 `main.js`, `manifest.json`, `styles.css`만 포함된 GitHub Release 초안을 만듭니다.
